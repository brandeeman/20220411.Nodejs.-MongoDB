Rest 원칙 6개
-Rest API 원칙을 잘 지키면 : Restful 하다고 함

1. Uniform Interface
- 하나의 URL로는 하나의 데이터를 가져와야함 
- URL 하나를 알면 둘을 알 수 있어야 함
- URL 이름짓기 관습을 잘 따라주세요 

(참고)
*URI란? 자료를 넘버링하고 분류하고 지칭하는 방법. URL과 비슷하지만 비슷하지만 조금 더 큰 의미
 
2. Client-server 역할 구분하기
-브라우저는 요청만 한다.
-서버는 응답만 한다.

3. Stateless
-요청1과 요청2는 의존성이 없어야 함

4. Cacheable
-캐싱 가능 : 크롬이 알아서 함 

*캐싱이란?
네이버를 방문하면 크롬 브라우저는 자동으로 자주 사용하는 이미지 파일, CSS 파일 등을 하드에 저장해놓습니다. 
별로 바뀔일 없는 네이버 로고나 아이콘 같은거요.
하드에 저장해놓고 네이버 방문할 때 네이버서버에 네이버 로고주세요~라고 요청하지 않고 하드에서 불러옵니다.
이 행위를 캐싱이라고 합니다. 

5. Layered System
-중요하지 않음
6. Code on Demand
-중요하지 않음
